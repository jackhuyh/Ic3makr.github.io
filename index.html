Linux Insides 笔记1 (Booting)

发表于 2016-09-03   |   分类于 书   |  

0x0 From bootloader to kernel

0x00 加电启动

加电后，CPU启动，运行在实模式下，并对寄存器初始化，跳转至Reset Vector，然后跳转至BIOS

0x01 BIOS

BIOS首先查看BIOS Configure，来确定boot的顺序

当要从一个硬盘（也可以是networks, optical media, USB based devices）boot时，BIOS首先会寻找boot sector，在MBR分区的硬盘上，boot sector存放在第一个扇区(共512字节)的前446个字节，然后开始执行

执行完boot sector code之后，BIOS会选择一个Bootloader，并将程序控制权交给它

Note: BIOS在ROM里

0x02 Bootloader

Bootloader是用来load system，和其他一些系统程序的，典型的有GRUB 2和syslinux

Bootloader会把Kernelload到内存里，然后开始执行kernel setup code(这个代码在kernel源码中/arch/x86/boot/header.S中可以找到)

0x03 Kernel

进入kernel setup code后

它主要做了以下的工作

  1. 确认所有段寄存器的值是一样的
  2. 建立栈
  3. 建立bss
  4. 转入C代码的main function (arch/x86/boot/main.c)

0x1 First steps in the kernel setup

prepare to move to proctected mode

0x10 Protected mode（保护模式）

Protected mode拥有更大的内存空间，并且适用于多进程的环境(防止进程间对彼此内存区域互相干扰)

segmentation（内存分段）

每个segment（分段）的segment descriptor（段描述符）存放在Global descriptor table (GDT)中，这个表的地址存放在GDTR寄存器中

在保护模式下寻找物理地址，首先要根据对应的段寄存器的index值来从GDT中找到对应的段描述符，段描述符中的base值即为基址，然后在根据该虚拟地址在该段终端偏移，两者相加即可得到线性地址，对于线性地址转换为实际物理地址，与分页机制有关，是后面的内容

0x11 ohter

  1. Copying boot parameters into the “zeropage”
  2. Console initialization
  3. Heap initialization
  4. CPU validation
  5. Memory detection
  6. Keyboard initialization
  7. Querying

这些过程均在/arch/x86/boot/main.c中可以找到

0x12 FAQ

Q: BIOS和kernel的区别是什么？
A: https://www.quora.com/What-is-the-difference-between-BIOS-and-kernel

xdctf-qual-2015-pwn500

发表于 2016-08-18   |   分类于 CTFs   |  

Environment

Local
Libc : libc6_2.19-0ubuntu6.6_amd64

Conclusion

分析的时候首先要把程序中用到的一些结构体的定义搞清楚，容易发现在show score中，会调用exam结构体的一个函数指针，看到这点比较容易想到利用UAF来修改这个函数指针完成一些事情。

但是这个UAF的bug我找了好久也没发现在哪里，还是基本功太不扎实了，后来还是参考了@fdu-sixstars的writeup

关键在于在free一个essay后，如何让这个存放在exam结构体中的essay的指针不被清零（这样就可以通过cheat来修改essay），之后再次take exam时，使得刚malloc出来的用来存放exam结构体的内存与之前的essay为同一块地方，这样只要通过修改之前的essay，就可以达到修改刚获得的exam结构体的内容，从而利用show score，来实现任意函数的调用

现在的问题就是如何使得essay指针不被清零，可以看看resit的代码(IDA反编译后)，程序是在判断exam结构体的essay的长度是否为0来决定要不要将essay置零

在exam结构体中的函数指针所指向的用来获得score的代码如下所示，通过判断score是否小于60来决定要不要将essay释放。

因此只要想办法使得我们输入essay后，在保证score低于60的情况下，仍能使得essay+4为0即可

程序首先会将输入的字符串通过fputs函数放到stream里

然后通过fread函数存放到essay中

因此只要在输入的essay的开头放上一个NULL字符，即会被fputs截断，这样就可以成功使得essay+4为0

到了这里后，接下去的思路就自己能很容易想出来了，利用printf函数的fsb漏洞，泄露栈上libc_start_main_ret的地址，然后通过libc-database搜索libc，在得到了system函数地址后，利用system("/bin/sh;junk")这样的trick就可以成功getshel了
