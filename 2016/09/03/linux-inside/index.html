Linux Insides 笔记1 (Booting)

发表于 2016-09-03   |   分类于 书   |  

0x0 From bootloader to kernel

0x00 加电启动

加电后，CPU启动，运行在实模式下，并对寄存器初始化，跳转至Reset Vector，然后跳转至BIOS

0x01 BIOS

BIOS首先查看BIOS Configure，来确定boot的顺序

当要从一个硬盘（也可以是networks, optical media, USB based devices）boot时，BIOS首先会寻找boot sector，在MBR分区的硬盘上，boot sector存放在第一个扇区(共512字节)的前446个字节，然后开始执行

执行完boot sector code之后，BIOS会选择一个Bootloader，并将程序控制权交给它

Note: BIOS在ROM里

0x02 Bootloader

Bootloader是用来load system，和其他一些系统程序的，典型的有GRUB 2和syslinux

Bootloader会把Kernelload到内存里，然后开始执行kernel setup code(这个代码在kernel源码中/arch/x86/boot/header.S中可以找到)

0x03 Kernel

进入kernel setup code后

它主要做了以下的工作

  1. 确认所有段寄存器的值是一样的
  2. 建立栈
  3. 建立bss
  4. 转入C代码的main function (arch/x86/boot/main.c)

0x1 First steps in the kernel setup

prepare to move to proctected mode

0x10 Protected mode（保护模式）

Protected mode拥有更大的内存空间，并且适用于多进程的环境(防止进程间对彼此内存区域互相干扰)

segmentation（内存分段）

每个segment（分段）的segment descriptor（段描述符）存放在Global descriptor table (GDT)中，这个表的地址存放在GDTR寄存器中

在保护模式下寻找物理地址，首先要根据对应的段寄存器的index值来从GDT中找到对应的段描述符，段描述符中的base值即为基址，然后在根据该虚拟地址在该段终端偏移，两者相加即可得到线性地址，对于线性地址转换为实际物理地址，与分页机制有关，是后面的内容

0x11 ohter

  1. Copying boot parameters into the “zeropage”
  2. Console initialization
  3. Heap initialization
  4. CPU validation
  5. Memory detection
  6. Keyboard initialization
  7. Querying

这些过程均在/arch/x86/boot/main.c中可以找到
